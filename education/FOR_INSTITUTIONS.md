# For Institutions — Lab and University Adoption

## How to adopt the framework at the organizational level.

---

## The Adoption Paradox

Institutions are cautious. That is their job. They protect
established standards. They verify before adopting. This is
CORRECT behavior — the framework itself endorses it: observe
and measure before adopting.

The framework does NOT ask institutions to adopt it on faith.
It asks institutions to TEST it and measure the results.

## The Three-Phase Adoption

### Phase 1 — The Pilot (One Lab, One Semester)

**What to do:**
- Select one lab or research group willing to try the framework
- Implement the five-folder structure for all AI-assisted work
- Train the team on domain classification (Stage 1: SORT)
- At the end of the semester, measure:
  - How many AI-generated outputs were initially misclassified?
  - Did the misclassification rate decrease over time?
  - Did the framework change how researchers interact with AI?
  - Did any publications benefit from the domain separation?
  - Did any researchers report improved clarity about AI outputs?

**What NOT to do:**
- Do not mandate the framework across the entire institution
- Do not change grading or tenure criteria based on the framework
- Do not report results before the pilot is complete
- Do not compare labs that use the framework against labs that
  do not (yet — that is Phase 2)

**Cost:** Zero. The framework is a directory structure and a set
of practices. No software to purchase. No hardware to install.
No consultants to hire.

### Phase 2 — The Comparison (Multiple Labs, One Year)

**What to do:**
- Expand to 3-5 labs across different departments
- Include at least one lab that does NOT use the framework
  (the control group)
- Measure over one year:
  - Research output quality (peer review acceptance rates)
  - Retraction rates (did framework labs have fewer retractions?)
  - AI-related incidents (did framework labs have fewer cases
    of boundary confusion?)
  - Researcher wellbeing (self-reported clarity and confidence)
  - Student performance (did students in framework labs produce
    more rigorous work?)

**The standard:** If framework labs show measurably better
outcomes on ANY of these metrics without measurably worse
outcomes on the others — the framework has earned Phase 3.

### Phase 3 — Institutional Integration

**What to do (only after Phase 2 shows positive results):**
- Develop institutional guidelines for AI-assisted research
  based on the framework
- Integrate domain classification into research reporting
  templates
- Include the five-folder structure in graduate student
  onboarding
- Add "AI domain classification" as a skill assessed in
  qualifying exams (not instead of existing skills — alongside)
- Publish the institutional experience as a case study for
  other institutions

**What NOT to do:**
- Do not make it bureaucratic — the framework should be
  LIGHTER than existing compliance, not heavier
- Do not turn domain classification into a checkbox exercise —
  it must remain a thinking skill
- Do not punish researchers who do not use the framework —
  the evidence should make adoption voluntary, not mandatory

## The Concerns Institutions Will Have

| Concern | Response |
|---------|----------|
| "This adds overhead to research" | It REMOVES the overhead of manually verifying every AI output. Sorting is faster than doubting. |
| "Our existing processes already handle this" | If so, the framework will produce no improvement in Phase 2, and you will have confirmed that your processes are sufficient. No harm done. |
| "The author has no credentials" | Correct. Evaluate the framework, not the author. The five-folder test costs nothing and takes one week. |
| "AI tools change too rapidly for a fixed framework" | The framework is tool-agnostic. It organizes OUTPUT, not tools. It works regardless of which AI system produced the output. |
| "Students will find workarounds" | Some will. The framework is designed to be EASIER than workarounds. Sorting is faster than pretending. |
| "This might slow down research" | Phase 2 will measure this directly. If it slows research without improving quality, drop it. |

## The Reporting Template

For institutions running a pilot, a standardized reporting
template:

```
FRAMEWORK PILOT REPORT
======================
Institution: [name]
Department: [name]
Duration: [start date — end date]
Participants: [number of researchers]
AI Systems Used: [which ones]

METRICS:
- Total AI outputs classified: [number]
- Initial misclassification rate: [%]
- Final misclassification rate: [%]
- Publications produced during pilot: [number]
- Publications that used domain labeling: [number]
- Researcher self-reported clarity (1-10): [average]
- AI-related incidents during pilot: [number and description]

QUALITATIVE OBSERVATIONS:
[What worked? What did not? What would you change?]

RECOMMENDATION:
[Continue / Expand / Discontinue / Modify]
```

This template enters the BRIDGE/ pipeline of the broader
framework — it IS a measurement that moves through incoming,
matching, and (if positive) confirmed.

## The Long Game

The framework does not need every institution to adopt it.
It needs ENOUGH institutions to test it that the data becomes
undeniable. If the framework improves research integrity
measurably and reproducibly across multiple institutions and
disciplines — adoption follows naturally.

The institutions that adopt first gain the advantage: their
researchers produce more trustworthy AI-assisted work, their
students graduate with a skill that will define the next era
of science, and their publications carry a verifiable chain
from hypothesis to confirmation.

The institutions that wait lose nothing — except time. The
framework will still be here when they are ready.
