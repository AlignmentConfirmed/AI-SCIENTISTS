## Design Philosophy — Diamond Resilience

The framework's design philosophy is called Diamond Resilience.

Diamond is not chosen as a metaphor for value or rarity. It is
chosen for three engineering properties:

- **Transparent** — domain separation lets you see verified
  and speculative content simultaneously without confusion
- **Structurally hard** — the five domains resist collapse
  because the separation IS the strength. Same element as
  charcoal (carbon), different structure, different properties.
- **Made of common material** — five folders on a laptop.
  The same tools everyone already has, structured differently.

This is a design philosophy, not a prediction. If the framework
works as intended, it produces research workflows that are
transparent, structurally resilient, and built from common tools.
Whether to adopt it is a choice. The framework is an open door.
