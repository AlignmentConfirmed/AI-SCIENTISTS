# Science Release — AI-Assisted Scientific Discovery Engine

## One core engine. Three disciplines. Unlimited sub-fields.

```
science/
├── engine/              ← The shared recursive convergence algebra
│   └── v1/core/            All disciplines plug into this
│
├── biology/             ← Life sciences applications
│   └── v1/                 Sub-fields: genetics, neuroscience, ecology...
│
├── chemistry/           ← Chemical sciences applications
│   └── v1/                 Sub-fields: molecular, organic, materials...
│
└── physics/             ← Physical sciences applications
    └── v1/                 Sub-fields: quantum, classical, astrophysics...
```

## The Principle

ONE engine powers ALL science. The engine provides:
- Exact rational arithmetic (zero rounding error)
- Recursive convergence (observe → hypothesize → test → measure → converge)
- Multi-dimensional state representation (the Delta)
- Structural separation of verified from speculative
- Validation gates (proposals must improve the measurement)

Each discipline provides:
- Domain-specific data encoding (how to represent biology/chemistry/physics as a Delta)
- Domain-specific operators (what transformations are meaningful in that field)
- Domain-specific tests (does the output match what a scientist in that field expects)

## Versioning

The engine and each discipline version INDEPENDENTLY:
- engine/v1 → engine/v2 (the algebra improves)
- biology/v1 → biology/v2 (the biological encoding improves)
- A discipline can stay on engine/v1 while the engine moves to v2
- A discipline can upgrade at its own pace
- No discipline is forced to adopt a new engine version

## Parallel Development

Each sub-field is an independent work stream. Multiple agents
or researchers can develop simultaneously:
- Agent 1 works on biology/v1/genetics/
- Agent 2 works on chemistry/v1/molecular/
- Agent 3 works on physics/v1/quantum/
- All three plug into engine/v1/core/
- No cross-contamination between work streams
- The directory structure IS the guardrail
