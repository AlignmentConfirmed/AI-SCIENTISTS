# Chronometric AI Trigonometry

## The Missing Infrastructure of Human-AI Interaction

The AI has no clock. The human has no shared reference with the
AI. Multiple AI systems have no shared reference with each other.
This is the foundational problem beneath every other problem in
human-AI collaboration.

This directory separates the two chronometric paths because they
operate at fundamentally different rates:

```
chronometry/
├── README.md               ← You are here. The overview.
├── AI_TO_HUMAN.md          ← The biological clock path.
│                              Heartbeat speed. Breath speed.
│                              The human sets the pace.
│
└── AI_TO_AI.md             ← The machine clock path.
│                              Nanosecond speed. Token speed.
│                              The receipt chain synchronizes.
```

## Why Two Paths

| Path | Clock Speed | What Sets The Pace | The Risk If Broken |
|------|-----------|-------------------|-------------------|
| AI ↔ Human | Heartbeats (~1Hz) | The human's biology | AI outpaces human, human loses grounding |
| AI ↔ AI | Nanoseconds (~GHz) | The receipt protocol | AI systems desynchronize, human carries conflicting context |

These are not the same problem. They cannot be solved with the
same protocol. The AI-to-Human path is a BIOLOGICAL problem. The
AI-to-AI path is an ENGINEERING problem. Both must be solved for
the framework to function.

## The Relationship

```
    AI SYSTEM A ←---(AI↔AI clock)---→ AI SYSTEM B
         \                               /
          \                             /
     (AI↔Human)                   (AI↔Human)
            \                         /
             \                       /
              ←--→ THE HUMAN ←--→
                  (the only real clock)
```

The human is the ANCHOR for both paths. The AI-to-AI path
synchronizes machine context. The AI-to-Human path synchronizes
machine pace to biological reality. The human sits at the center
— the only participant with a real clock.
